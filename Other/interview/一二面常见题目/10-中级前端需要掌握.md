> 订阅发布模式

订阅发布模式，使用起来就是像 JQ 里 on/off/trigger 触发事件，在共享的访问空间存储 hashMap，键是一个‘事件名’，值是回调函数组成数组。

在 JQ 中有 on,off 移除事件，emit 触发事件（trigger）

```
 class Event{
     constructor(){
         this.eventMap = {}
     }

     on(eventName,fn){
        let eventsList = this.eventMap[eventName]
        eventsList = eventsList ? eventsList : []
        eventsList.push(fn)
     }

     off(eventName,fn){
         let fnIndex = this.eventMap[eventName].indexOf(fn)
         this.eventMap[eventName].splice(fnIndex,1)
     }

     emit(eventName,...params){
         this.eventMap[eventName].forEach(fn => {
             fn(...params)
         })
     }

 }
```

> 单例模式

单例模式: 就是一个类只有一个实例，并且提供一个访问它的访问点

```
      function Animal() {
            this.type = 'animal'
            this.instance = null
        }

        Animal.getInstance = function () {
            if(!this.instance){
                this.instance = new Animal()
            }
            return this.instance
        }

        var animal1 =  Animal.getInstance()
        var animal2 =  Animal.getInstance()

        console.log(animal1 === animal2)
```

那其实这个用 JS 强行实现一个单例模式，可能 JAVA 这些，以类为中心语言。通过类来创建对象是很自然的事情。

## 2.React 和 Vue 的区别 《不黑不吹聊聊前端框架》

- 1. 组件的理解和分类
    + 组件分为展示型、交互型、功能型
    + JSX和模板相比，最大的特点是灵活。因为JS所具备的特点，JSX都可以有,JSX对功能型组件的使用比Vue强大很多
    + 模板的好处在展示型的组件写起来比较爽，因为模板强制要求你写尽可能少的逻辑在template
- 2. 变化侦测机制
    + 但是如果内容变化，侦测机制不太一样。分为pull和push2种
    + react里setState，就是react不知道哪个数据发生变化，它需要一个信号，一旦变化，就地虚拟dom做diff对比，所以是一个比较暴力的方式。那之所以可以这么做是因为JS现在快。
    + vue是监听每一个数据的变化，可以更加细粒度的知道哪个数据需要更新。
    + 但是实际上，react和vue都还各自有一些优化技巧。比如react在shouldUpdateComponet里可以做优化,
    而Vue2.0之后也是2种方式都借鉴。在组件级别，监听哪个组件里有数据变化。在组件内部用虚拟dom去做对比来更新。
- 3. 状态管理
    + 总体上，前端的数据状态管理方案可能还没有统一，有2个比较相似的方案
    + redux和mobx
    + 尤大是说认为简单的项目没必要抽象出太多层，比如redux/react-redux/react-thunk/react-promsie之类的，如果项目特别复杂，可以考虑用rx.js
- 4. 路由
    + 区别并不是特别
    + react-router4改动会大一点。改动主要特点就是去中心化。
    + 使用组件本身去做路由，很好利用了JSX的灵活性，一点缺点就是路由映射表也是可以帮助我们理解项目结构
- 5. CSS方案
    + 传统的css方案，就是跟js完全分离
    + react是 css in js比较激进的做法
    + vue比较这种，用过vue文件组织js和css代码

## 3.为什么选择Vue,Vue和传统项目相比，有什么区别

- 学习成本。Vue的上手速度和学习成本更低
- 项目复杂度。整体而言，项目也不算很巨大的项目，可能算是中型项目，我觉得Vue更适合我们这个场景复杂度。
（当然，也不是说React特别适合大型项目，因为我觉得如果项目真的很复杂，那可能用什么框架都不能避免它的复杂）
- 老大主导。我们公司老大，他最开始是主导整个项目的前后端的嘛，但是他只有Vue的使用经验。

和传统的JQ相比，我认为好处有3点:

- 数据驱动的方式写代码，肯定开发体验比操作DOM更便捷，开发速度更快
- 促进组件化/规范化，天生的.vue文件，你就很自然去做组件化的拆分，而且框架帮做了一些规范。上面是template，数据定义在data里，方法都写在method里。那所有Vue开发的人，都是这样写。
- 虚拟Dom，可以提升性能



## 4.前后端分离的原理及意义？

意义:

- 前端做业务逻辑，后端处理数据和接口，各自维护自己的代码。可以让前后端代码解耦
- 减轻服务器的压力，单页应用并不是每一个页面变化都需要请求接口。
- 增强用户体验。比如从首页跳到详情页，节省掉dns/http/接口响应/渲染页面

原理:

histrory 和 hash

- 1.能执行一个打开的动作
- 2.要有历史记录操作单，能前进后退
